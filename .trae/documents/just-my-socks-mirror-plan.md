# Just My Socks Mirror 实施计划

## 目标

基于 Vercel 实现一个 Just My Socks Mirror 项目，包含：

1. `/members/*` 到可配置上游 `/members/*` 的透明代理能力
2. 上游地址通过 ENV 配置，默认使用 `https://jjsubmarines.com`
3. `/` 首页，用于输入 `service` 与 `id`，展示订阅链接、流量进度和重置日期
4. 订阅链接参数实时切换：

   * 排除 Shadowsocks → `noss=1`

   * 排除 Vmess → `novmess=1`

   * 排除 IP 服务器 → 使用 `usedomains=1`
5. 默认勾选“排除 IP 服务器”
6. 页面上的订阅地址必须指向当前镜像站自己的 `/members/getsub.php`
7. 流量信息通过当前镜像站自己的 `/members/getbwcounter.php` 获取，再由代理转发到上游

## 实现思路

### 一、项目基础结构

* 采用适合部署到 Vercel 的前端全栈方案，优先选择一个实现轻量代理和首页 UI 都方便的结构。

* 初始化项目基础配置，保证本地开发与 Vercel 部署路径一致。

* 补齐最少必要脚本：开发、构建、类型检查、静态检查。

* 预留环境变量读取逻辑，确保本地与线上都可以覆盖默认上游地址。

### 二、代理能力设计

目标：所有 `/members/*` 请求都由当前站点转发到“可配置上游”的 `/members/*`。

上游规则：

* 默认上游：`https://jjsubmarines.com`

* 运行时允许通过 ENV 覆盖，例如：`JMS_UPSTREAM_BASE_URL`

* 实际代理目标格式：`{UPSTREAM_BASE_URL}/members/{path}`

具体步骤：

1. 实现一个 Vercel 可部署的服务端路由，匹配 `/members/:path*`
2. 读取环境变量中的上游基础地址；若未配置则回退到 `https://jjsubmarines.com`
3. 将原始请求路径、查询参数完整拼接到目标地址
4. 透传请求方法、查询字符串、必要请求头
5. 将上游响应状态码、响应头、响应体尽量原样返回
6. 处理文本、JSON、订阅内容等不同响应类型，避免破坏原始返回内容
7. 对异常情况返回明确错误信息，避免页面无反馈

需要覆盖的典型请求：

* `/members/getsub.php?service=...&id=...`

* `/members/getbwcounter.php?service=...&id=...`

### 三、首页交互设计

#### 1. 参数输入流程

* 用户访问 `/` 且缺少 `service` 或 `id` 时，显示模态框

* 模态框内提供两个输入项：`service`、`id`

* 提交后跳转到 `/?service={service}&id={id}`

* 若 URL 已存在这两个参数，则直接展示主页面内容

#### 2. 页面内容

页面包含：

* 标题

* 一行简短介绍

* 流量使用情况卡片

* 订阅链接卡片

* 参数切换区域

#### 3. 订阅链接逻辑

订阅地址基础格式：
`{当前站点}/members/getsub.php?service={service}&id={id}`

在此基础上按复选框实时追加参数：

* 排除 Shadowsocks → `noss=1`

* 排除 Vmess → `novmess=1`

* 排除 IP 服务器 → `usedomains=1`

说明：这里按你的需求将“排除 IP 服务器”落实为“使用域名代替 IP”，即追加 `usedomains=1`。这会让订阅结果优先使用域名节点，而不是 IP 节点。

页面行为：

* 默认勾选“排除 IP 服务器”

* 用户切换任一复选框时，订阅链接实时更新

* 提供复制按钮，复制当前完整订阅链接

### 四、流量信息展示设计

数据来源：

* 前端请求 `/members/getbwcounter.php?service={service}&id={id}`

* 由代理转发至上游后返回 JSON

处理逻辑：

1. 读取 `monthly_bw_limit_b`
2. 读取 `bw_counter_b`
3. 计算：

   * 已用流量

   * 剩余流量

   * 使用百分比
4. 将字节格式化为 GB 或更适合的单位
5. 根据 `bw_reset_day_of_month` 计算下一次重置日期
6. 重置日期按 Los Angeles 时区语义展示，例如 `2026-04-16`

页面展示：

* 进度条：显示已用 / 总量

* 文本：例如 `292.071 GB / 1000 GB`

* 文本：剩余额度

* 文本：下次刷新日期

* 加载中、获取失败状态

### 五、日期计算规则

* 以当前时间为基础

* 按 Los Angeles 时区判断当前日期

* 若当前日期未超过重置日，则本月重置日为下一次刷新

* 若当前日期已超过重置日，则取下个月的重置日

* 输出稳定的 `YYYY-MM-DD` 格式

### 六、ENV 设计

计划使用环境变量：

* `JMS_UPSTREAM_BASE_URL`

规则：

* 未配置时默认值为 `https://jjsubmarines.com`

* 配置时要求为完整 URL，不带 `/members` 后缀也能正确拼接

* 实现时会统一去除末尾 `/`，避免出现双斜杠

部署含义：

* Vercel 项目可直接在环境变量里切换上游域名

* 若未来官方域名变更，无需改代码，只需改 ENV

### 七、UI 细节

* 整体保持简单、清晰、移动端可用

* 模态框优先展示，避免用户在缺少参数时看到空页面

* 流量卡片与订阅卡片分区明确

* 复制按钮在成功后给出轻量反馈

* 对接口错误、参数缺失、无效返回值做用户可理解的提示

### 八、实施顺序

1. 初始化 Vercel 项目骨架与依赖
2. 实现带 ENV 上游配置的 `/members/*` 代理
3. 实现首页基础布局与参数读取
4. 实现参数输入模态框与 URL 跳转
5. 实现订阅链接实时拼接与复制功能
6. 实现 `getbwcounter.php` 数据请求、进度条与重置日期计算
7. 补充加载态、错误态、空态
8. 补充环境变量默认值与读取封装
9. 进行本地构建、类型检查、静态检查并修复问题

## 验收标准

* 访问 `/members/getsub.php?...` 时能正确代理并返回订阅内容

* 访问 `/members/getbwcounter.php?...` 时能正确代理并返回 JSON

* 未设置 ENV 时，代理默认走 `https://jjsubmarines.com`

* 设置 ENV 后，代理会改为使用指定上游

* 访问 `/` 时，无参数会弹出输入模态框

* 输入 `service` 和 `id` 后会跳转到带查询参数的页面

* 页面能展示流量进度、已用/总量、重置日期

* 页面能展示当前镜像站的订阅链接

* 勾选项切换后，订阅链接实时变化

* 默认勾选“排除 IP 服务器”

* 复制按钮可复制当前订阅链接

* 项目可在 Vercel 正常部署

## 当前假设

* 当前仓库为空或接近空仓，可直接按新项目初始化

* “排除 IP 服务器”按 `usedomains=1` 实现

* 首页无需登录体系，仅依赖 URL 中的 `service` 与 `id`

* 只需要一个简洁单页，不额外扩展账户管理等能力

* 默认上游固定为 `https://jjsubmarines.com`

## 后续执行时会重点验证

* 代理响应头是否需要过滤部分 hop-by-hop headers

* 订阅内容是否需要特别处理 content-type

* 上游 ENV 拼接 `/members` 时是否会出现路径重复或双斜杠

* Los Angeles 时区日期计算在月末场景是否稳定

* Vercel 路由匹配方式是否与最终项目结构完全一致
